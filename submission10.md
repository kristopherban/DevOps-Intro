# Lab 10 — Cloud Computing Fundamentals

## Task 1 — Artifact Registries

### Какие сервисы есть у провайдеров

- **AWS ECR** — хранилище для Docker-образов. Используется вместе с ECS или EKS.
- **GCP Artifact Registry** — хранит Docker-образы, а ещё пакеты для npm, Maven, Python.
- **Azure ACR** — хранилище для Docker-образов и Helm-чартов.

### Сравнение

| Что сравниваем | AWS ECR | GCP Artifact Registry | Azure ACR |
|----------------|---------|----------------------|-----------|
| Что можно хранить | Docker | Docker, npm, Maven, Python | Docker, Helm |
| Проверка на уязвимости | Есть | Есть | Есть |
| Копирование в другие регионы | Да | Да | Только в дорогой версии |
| Интеграция с CI/CD | Да | Да | Да |
| Цена | За объём + трафик | За объём + трафик | За объём + трафик |

### Что лучше для нескольких облаков

Если проект использует разные облачные платформы, лучше взять GCP Artifact Registry. Почему:
- Поддерживает больше типов файлов
- Меньше привязан к конкретному провайдеру

Но вообще для мультиоблака часто используют отдельные инструменты типа Harbor.

## Task 2 — Serverless Computing

### Какие сервисы есть у провайдеров

- **AWS Lambda** — запускает код по событиям (например, по HTTP запросу или при загрузке файла в S3)
- **GCP Cloud Functions** — то же самое, но от Google. Ещё есть Cloud Run для контейнеров.
- **Azure Functions** — серверлес от Microsoft

### Сравнение

| Что сравниваем | AWS Lambda | GCP Cloud Functions | Azure Functions |
|----------------|------------|---------------------|-----------------|
| Языки | Python, JS, Java, Go, C# | Python, JS, Go, Java | C#, JS, Python, Java |
| Макс. время работы | 15 минут | 9 минут | 10 минут |
| Холодный старт | 100-300 мс | 100-400 мс | 200-500 мс |
| Цена за 1 млн вызовов | $0.20 | $0.40 | $0.20 |

### Что выбрать для REST API

Для REST API лучше всего подходит AWS Lambda + API Gateway. Причины:
- Хорошая работа с HTTP запросами
- Быстрый холодный старт
- Много примеров в интернете

### Плюсы и минусы серверлес

**Плюсы:**
- Не нужно настраивать серверы
- Код сам масштабируется под нагрузку
- Платишь только когда код работает

**Минусы:**
- Первый запрос после простоя может быть медленным
- Нельзя запускать код дольше 15 минут
- Сложно отлаживать на локальном компьютере
- Привязка к конкретному облаку

### Итог

Serverless подходит для REST API, обработки событий и фоновых задач. Не подходит для долгих вычислений и приложений, которым нужны стабильно низкие задержки.
